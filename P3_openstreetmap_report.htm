Other ideas about the datasets

We have all information in google maps in current world. So we can use google API to load the correct address for each node (and whatever other information available), but if openstreetmap dataset has information for same node, Its recommended that we verify atleast partial match between OSM file data and google API data, if not matching any info we must verify what is correct. This will be manual or should be dependent on any third dataset.

We have also seen that most of the amenities dont have address, so to improve data, we can bring address, phone and all other information from google maps and load it.
